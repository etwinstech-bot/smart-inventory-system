🔐 Authentication & Authorization

User Roles

Role User Management Product Management Staff Creation View Logs
Superadmin ✅ Full CRUD ✅ Full CRUD ✅ Any role ✅ Yes
Admin ❌ No ✅ Full CRUD ✅ Staff/User only ✅ Yes
Staff ❌ No ✅ View only ❌ No ✅ Yes
User ❌ No ✅ View only ❌ No ❌ No

Authentication Flow

  1. User logs in with email/password
  2. System hashes password with bcrypt and compares
  3. On success, JWT token is issued (24-hour expiry)
  4. Token stored in localStorage (client-side)
  5. Every API request includes token in Authorization header
  6. Middleware verifies token and attaches user info to request

📊 Data Models

User Schema

  • name (String, required)
  • email (String, required, unique)
  • password (String, hashed, required)
  • role (String, enum: superadmin|admin|staff|user)
  • status (String, enum: active|inactive|suspended)
  • createdAt (Date, default: now)
  • updatedAt (Date, auto-updated)
  • lastLogin (Date, tracks user activity)

Product Schema

  • name (String, required)
  • category (String)
  • price (Number)
  • quantity (Number)
  • createdAt (Date)
  • updatedAt (Date)

Log Schema

  • user (ObjectId, user reference)
  • action (String, description)
  • timestamp (Date, auto-generated)

🔧 API Endpoints

Authentication Endpoints

  • POST /api/auth/register - Register new user (setup)
  • POST /api/auth/login - Login and get JWT token
  • POST /api/auth/create-user - Create user (admin/superadmin only)

User Management Endpoints

  • GET /api/users - Get all users (superadmin only)
  • GET /api/users/:id - Get single user (superadmin only)
  • PUT /api/users/:id - Update user details (superadmin only)
  • PUT /api/users/:id/role - Change user role (superadmin only)
  • PUT /api/users/:id/password - Reset password (superadmin only)
  • DELETE /api/users/:id - Delete user (superadmin only)
  • GET /api/users/stats/overview - Get user statistics (superadmin only)

Product Endpoints

  • GET /api/products - Get all products (authenticated users)
  • GET /api/products/:id - Get single product (authenticated users)
  • POST /api/products/add - Add product (admin/superadmin)
  • PUT /api/products/:id - Update product (admin/superadmin)
  • DELETE /api/products/:id - Delete product (admin/superadmin)

Log Endpoints

  • GET /api/logs - Get system logs (authenticated users)

🔒 Security Considerations

  • All passwords hashed with bcrypt (10 salt rounds)
  • JWT tokens expire after 24 hours
  • Role-based access control enforced at middleware level
  • Admin cannot create admin/superadmin accounts (enforced in code)
  • Superadmin-only destructive operations (user deletion)
  • All API requests require valid JWT token
  • Sensitive data (password) excluded from API responses
